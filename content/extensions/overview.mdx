# 扩展中心

<ExtendedDescription>
  扩展（Extensions）是 App Inventor 的强大功能，让您可以添加内置组件不支持的功能。通过扩展，您可以访问更多硬件特性、使用第三方服务、创建自定义功能。
</ExtendedDescription>

## 什么是扩展？

<AccordionGroup>
  <Accordion title="扩展的定义" defaultOpen>
    扩展是额外的代码组件，以 `.aix` 文件格式提供。它们由 MIT 或第三方开发者创建，可以像内置组件一样在 App Inventor 中使用。
  </Accordion>
  <Accordion title="为什么使用扩展？">
    - 添加内置不支持的功能
    - 更强大的硬件控制
    - 访问第三方服务和API
    - 更高的性能和更好的兼容性
  </Accordion>
  <Accordion title="如何安装扩展？">
    1. 下载 `.aix` 文件
    2. 在 App Inventor 中，点击 **扩展 (Extensions)**
    3. 点击 **导入扩展 (Import Extension)**
    4. 上传下载的 `.aix` 文件
    5. 从组件面板中拖出新的扩展组件
  </Accordion>
</AccordionGroup>

## 常见问题

<AccordionGroup>
  <Accordion title="No Such Class 错误" defaultOpen>
    这是最常见的扩展错误。解决方法：

    1. **重启 Companion**：完全关闭AI2 Companion，从最近任务中滑出，重新打开
    2. **重新导入扩展**：删除项目中的扩展，重新下载并导入
    3. **在 Screen1 添加扩展**：有些扩展需要在 Screen1 中初始化
    4. **构建 APK 测试**：Companion 环境的问题，构建 APK 后通常能解决
  </Accordion>
  <Accordion title="扩展冲突">
    多个扩展可能导致 DX Execution Failed 错误：

    1. 移除所有扩展，逐个添加测试
    2. 查找冲突的扩展组合
    3. 考虑使用功能相似的其他扩展
  </Accordion>
  <Accordion title="Android 版本兼容性">
    不同 Android 版本有不同的权限要求：

    - **Android 12+**：需要精确闹钟权限
    - **Android 13+**：需要通知权限
    - **Android 14+**：存储限制更严格

    在使用扩展功能前，需要申请相应权限。
  </Accordion>
  <Accordion title="扩展安全性">
    使用扩展时的安全考虑：

    - 只从可信来源下载（MIT官网、知名开发者）
    - 检查所需权限是否合理
    - 查看源代码（如果是开源项目）
    - 定期更新到最新版本
  </Accordion>
</AccordionGroup>
