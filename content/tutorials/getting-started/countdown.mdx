# App Inventor 2 倒计时教程详解

> 本文详细讲解 App Inventor 2 中倒计时的多种实现方式，适合初学者到进阶用户。

---

## 一、倒计时用到的核心组件：计时器（Clock）

计时器（Clock）是 App Inventor 2 中实现倒计时的核心组件，位于**传感器**分类下。

**关键属性：**

| 属性 | 说明 | 默认值 |
|------|------|--------|
| 计时间隔（TimerInterval） | 每隔多少毫秒触发一次计时事件 | 1000ms（1秒） |
| 启用计时（TimerEnabled） | 是否启动计时器 | 勾选（启用） |
| 一直计时（TimerAlwaysFires） | App后台/屏幕关闭时是否继续计时 | 勾选（继续） |

**核心事件：**
- **计时（Timer）**：每隔"计时间隔"时间触发一次，倒计时逻辑写在这里

---

## 二、最简单的60秒倒计时

### 实现思路

1. 设置计时器间隔为 **1000毫秒**（1秒）
2. 定义全局变量 `剩余秒数 = 60`
3. 每次计时事件触发时，`剩余秒数 - 1`
4. 将剩余秒数显示在标签上
5. 当 `剩余秒数 = 0` 时，停止计时器，提示倒计时结束

### 代码块逻辑

```
初始化全局变量 剩余秒数 = 60

当 计时器1.计时 时
  设置 全局变量 剩余秒数 = 全局变量 剩余秒数 - 1
  设置 标签1.文本 = 合并字符串("剩余：", 全局变量 剩余秒数, "秒")
  如果 全局变量 剩余秒数 = 0 则
    设置 计时器1.启用计时 = 假
    调用 对话框1.显示消息对话框("倒计时结束！", "提示", "确定")
```

### 注意事项

- 计时器间隔设为 `1000`（毫秒），即每秒触发一次
- 变量从 60 开始递减，到 0 时停止
- 停止计时器用：`设置 计时器1.启用计时 = 假`

## 三、带开始/暂停/重置的倒计时

实际应用中，倒计时通常需要支持**开始、暂停、重置**三个操作。

### 界面设计

- 标签：显示剩余时间
- 按钮1：开始/暂停（切换）
- 按钮2：重置
- 计时器：间隔1000ms

### 代码块逻辑

```
初始化全局变量 剩余秒数 = 60
初始化全局变量 是否运行中 = 假

当 按钮1.被点击 时
  如果 全局变量 是否运行中 = 假 则
    设置 计时器1.启用计时 = 真
    设置 全局变量 是否运行中 = 真
    设置 按钮1.文本 = "暂停"
  否则
    设置 计时器1.启用计时 = 假
    设置 全局变量 是否运行中 = 假
    设置 按钮1.文本 = "继续"

当 按钮2.被点击 时（重置）
  设置 计时器1.启用计时 = 假
  设置 全局变量 剩余秒数 = 60
  设置 全局变量 是否运行中 = 假
  设置 标签1.文本 = "60秒"
  设置 按钮1.文本 = "开始"

当 计时器1.计时 时
  设置 全局变量 剩余秒数 = 全局变量 剩余秒数 - 1
  设置 标签1.文本 = 合并字符串(全局变量 剩余秒数, "秒")
  如果 全局变量 剩余秒数 = 0 则
    设置 计时器1.启用计时 = 假
    设置 全局变量 是否运行中 = 假
    设置 按钮1.文本 = "开始"
    调用 对话框1.显示消息对话框("时间到！", "提示", "确定")
```

## 四、分钟级倒计时（时:分:秒格式）

### 实现思路

将总秒数转换为 `时:分:秒` 格式显示。

```
初始化全局变量 总秒数 = 3600  （1小时 = 3600秒）

定义过程 格式化时间(秒数)
  初始化局部变量 小时 = 商(秒数, 3600)
  初始化局部变量 分钟 = 商(余数(秒数, 3600), 60)
  初始化局部变量 秒 = 余数(秒数, 60)
  返回 合并字符串(
    如果 小时 < 10 则 合并("0", 小时) 否则 小时,
    ":",
    如果 分钟 < 10 则 合并("0", 分钟) 否则 分钟,
    ":",
    如果 秒 < 10 则 合并("0", 秒) 否则 秒
  )

当 计时器1.计时 时
  设置 全局变量 总秒数 = 全局变量 总秒数 - 1
  设置 标签1.文本 = 调用 格式化时间(全局变量 总秒数)
  如果 全局变量 总秒数 = 0 则
    设置 计时器1.启用计时 = 假
    调用 对话框1.显示消息对话框("倒计时结束！", "提示", "确定")
```

**关键数学运算：**
- `商(3661, 3600)` = 1（小时）
- `余数(3661, 3600)` = 61，`商(61, 60)` = 1（分钟）
- `余数(61, 60)` = 1（秒）
- 结果：`01:01:01`

## 五、定点倒计时（倒计时到指定时间）

有时需要倒计时到某个具体时间点，比如"距离考试还有多少时间"。

### 实现思路

1. 获取当前时间戳（毫秒）
2. 设置目标时间戳
3. 差值即为剩余毫秒数
4. 每秒更新显示

```
初始化全局变量 目标时间文本 = "2026-06-07 09:00:00"

当 计时器1.计时 时
  初始化局部变量 当前毫秒 = 调用 计时器1.获取毫秒数(调用 计时器1.现在)
  初始化局部变量 目标毫秒 = 调用 计时器1.获取毫秒数(
    调用 计时器1.将文本转换为日期时间(全局变量 目标时间文本)
  )
  初始化局部变量 剩余毫秒 = 目标毫秒 - 当前毫秒
  
  如果 剩余毫秒 > 0 则
    初始化局部变量 剩余秒 = 商(剩余毫秒, 1000)
    初始化局部变量 天 = 商(剩余秒, 86400)
    初始化局部变量 时 = 商(余数(剩余秒, 86400), 3600)
    初始化局部变量 分 = 商(余数(剩余秒, 3600), 60)
    初始化局部变量 秒 = 余数(剩余秒, 60)
    设置 标签1.文本 = 合并字符串(天, "天", 时, "时", 分, "分", 秒, "秒")
  否则
    设置 标签1.文本 = "时间到！"
    设置 计时器1.启用计时 = 假
```

## 六、多任务倒计时

如果需要同时运行多个倒计时，有两种方案：

### 方案一：多个计时器组件（推荐）

每个倒计时使用独立的计时器组件，互不干扰，逻辑清晰。

### 方案二：一个计时器管理多个任务

用一个计时器，定义多个全局变量分别记录各任务的剩余时间：

```
初始化全局变量 任务A剩余 = 30
初始化全局变量 任务B剩余 = 60

当 计时器1.计时 时
  如果 全局变量 任务A剩余 > 0 则
    设置 全局变量 任务A剩余 = 全局变量 任务A剩余 - 1
    设置 标签A.文本 = 合并字符串("任务A：", 全局变量 任务A剩余, "秒")
  
  如果 全局变量 任务B剩余 > 0 则
    设置 全局变量 任务B剩余 = 全局变量 任务B剩余 - 1
    设置 标签B.文本 = 合并字符串("任务B：", 全局变量 任务B剩余, "秒")
```

## 七、常见问题

### Q1：倒计时不准确怎么办？

计时器间隔设为1000ms，但实际执行可能有几毫秒误差。如果需要精确倒计时，建议：
- 记录**开始时间戳**
- 每次计时事件用 `当前时间 - 开始时间` 计算实际经过时间
- 而不是简单地每次减1

### Q2：App切到后台倒计时还在跑吗？

取决于**"一直计时"**属性：
- 勾选（默认）：后台和锁屏都继续计时
- 不勾选：只有App在前台时才计时

### Q3：屏幕跳转后倒计时怎么保持？

- 使用**"一直计时"**属性
- 或将倒计时逻辑放在主屏幕，通过全局变量传递状态

### Q4：如何让倒计时结束时播放声音？

在倒计时结束的判断中，调用**音效**组件的播放方法：
```
调用 音效1.播放
```
