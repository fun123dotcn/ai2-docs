## 一、DynamicComponents 动态组件扩展 ⭐⭐⭐⭐⭐

### 1.1 为什么需要动态组件？

| 场景 | 问题 | DynamicComponents解决方案 |
|------|------|---------------------------|
| 聊天消息列表 | 消息数量不固定 | 动态创建消息组件 |
| 表单生成器 | 表单项由用户决定 | 运行时动态添加输入框 |
| 购物车 | 商品数量不确定 | 动态生成商品卡片 |
| 数据展示 | 数据行数变化 | 根据数据量动态创建行 |

### 1.2 扩展安装

**下载地址**：https://community.appinventor.mit.edu/t/dynamic-components/116586

**安装步骤**：

1. 下载 `DynamicComponents.aix` 文件
2. 打开 App Inventor 2 → 点击左侧「扩展」
3. 点击「导入扩展」→ 选择下载的 `.aix` 文件
4. 拖拽 `DynamicComponents1` 到界面上

### 1.3 核心积木块

```
┌─────────────────────────────────────────────────────┐
│ DynamicComponents1                                   │
├─────────────────────────────────────────────────────┤
│ 🔧 创建组件(组件名称, 组件类型)                      │
│ 🔧 删除组件(组件名称)                               │
│ 🔧 获取组件(组件名称) → 组件                         │
│ 🔧 获取组件列表() → 列表                             │
│                                                     │
│ 📐 设置属性(组件, 属性名称, 属性值)                 │
│ 📐 获取属性(组件, 属性名称) → 文本                   │
│                                                     │
│ 🎯 绑定事件(组件, 事件名称, 过程)                    │
└─────────────────────────────────────────────────────┘
```

### 1.4 实战案例：动态聊天消息

**目标**：根据消息列表动态创建聊天气泡

**界面设计**：

```
Screen1
├── DynamicComponents1
├── VerticalScrollArrangement1（聊天容器）
│   └──（动态创建）
└── TextBox1（输入框）
```

**逻辑代码**：

```
┌─ 全局变量 ─────────────────────────────────────────┐
│ 初始化全局 消息列表 = [["user", "你好"],          │
│                        ["bot", "您好，有什么帮助？"]]
└────────────────────────────────────────────────────┘

┌─ 初始化聊天 ──────────────────────────────────────┐
│ 当 Screen1.初始化 时
│ ├── 循环 for each 消息 in 全局消息列表
│ │   调用 添加消息气泡(获取列表项(全局消息列表, 消息))
│ └── 设置 VerticalScrollArrangement1.Height = -2
└────────────────────────────────────────────────────┘

┌─ 添加消息气泡 ────────────────────────────────────┐
│ 定义 过程 添加消息气泡(消息内容, 发送者)
│ ├── 如果 发送者 = "user" 则
│ │   设置 组件名称 = "用户消息" & 随机数(1, 10000)
│ │   调用 DynamicComponents1.创建组件(组件名称, "HorizontalArrangement")
│ │   调用 DynamicComponents1.设置属性(获取组件(组件名称), "Width", -2)
│ │   调用 DynamicComponents1.设置属性(获取组件(组件名称), "AlignHorizontal", 2)  // 右对齐
│ │
│ │   // 创建气泡标签
│ │   调用 DynamicComponents1.创建组件(组件名称 & "_气泡", "Label")
│ │   调用 DynamicComponents1.设置属性(获取组件(组件名称 & "_气泡"), "Text", 消息内容)
│ │   调用 DynamicComponents1.设置属性(获取组件(组件名称 & "_气泡"), "BackgroundColor", #E3F2FD)
│ │   调用 DynamicComponents1.设置属性(获取组件(组件名称 & "_气泡"), "TextColor", #000000)
│ │   调用 DynamicComponents1.设置属性(获取组件(组件名称 & "_气泡"), "Padding", "8,4,8,4")
│ │
│ │   // 添加到容器
│ │   调用 DynamicComponents1.添加组件到容器(
│ │       VerticalScrollArrangement1, 获取组件(组件名称))
│ │
│ └── 否则
│     // 机器人消息逻辑（类似，左对齐，蓝色气泡）
│     设置 组件名称 = "机器人消息" & 随机数(1, 10000)
│     ...（类似逻辑，背景色设为 #BBDEFB）
│ └── 条件结束
└────────────────────────────────────────────────────┘
```

### 1.5 动态组件属性速查

| 属性名 | 说明 | 示例值 |
|--------|------|--------|
| Width | 宽度 | -1=自动, -2=填充父级, 数字=固定像素 |
| Height | 高度 | -1=自动, -2=填充父级, 数字=固定像素 |
| Text | 文本内容 | "你好" |
| BackgroundColor | 背景颜色 | #FFFFFF, #E3F2FD |
| TextColor | 文字颜色 | #000000 |
| FontSize | 字体大小 | 14 |
| Padding | 内边距 | "8,4,8,4" |
| AlignHorizontal | 水平对齐 | 1=左, 2=中, 3=右 |
| AlignVertical | 垂直对齐 | 1=上, 2=中, 3=下 |

┌─ 提取邮箱 ─────────────────────────────────────────┐

│ ├── 设置 邮箱结束 = 全文.查找("@")
│ ├── 设置 邮箱结束 = 邮箱结束 + 全文.子文本(邮箱结束, ).查找(" ")
│ └── 返回 全文.子文本(邮箱起始, 邮箱结束)

## 五、BLE低功耗蓝牙扩展 ⭐⭐⭐⭐

### 5.1 BLE vs 传统蓝牙

| 对比项 | 传统蓝牙 | BLE蓝牙 |
|--------|---------|---------|
| 功耗 | 较高 | 低 |
| 连接距离 | 10-30米 | 50-100米 |
| 配对 | 需要配对 | 无需配对 |
| 适用场景 | 音频传输 | 传感器、物联网 |

### 5.2 扩展安装

**官方扩展**：MIT官方提供 BLE 组件（内置，无需单独安装）

### 5.3 实战案例：连接蓝牙手环获取步数

```

└────────────────────────────────────────────────────┘

│       标签="今日心率",

## 七、扩展开发入门

### 7.1 开发环境准备

```
必需工具：

2. Android SDK
3. App Inventor源码

### 7.2 扩展基本结构

```java
// HelloWorldExtension.java
package com.example.helloworld;

import com.google.appinventor.components.annotations.*;
import com.google.appinventor.components.common.*;
import com.google.appinventor.components.runtime.*;

@SimpleComponent(
    description = "一个简单的示例扩展",
    category = ComponentCategory.EXTENSION,
    iconName = "images/hello.png"
)
public class HelloWorldExtension extends AndroidNonvisibleComponent {
    
    public HelloWorldExtension(ComponentContainer container) {
        super(container.$form());
    }
    
    @SimpleFunction(
        description = "显示问候消息"
    )
    public String SayHello(String name) {
        return "你好，" + name + "！";
    }
    
    @SimpleEvent(
        description = "当问候完成时触发"
    )
    public void HelloCompleted(String message) {
        EventDispatcher.dispatchEvent(this, "HelloCompleted", message);
    }
}
```

### 7.3 编译扩展

```bash
# 1. 克隆App Inventor源码
git clone https://github.com/mit-cml/appinventor-sources.git

# 2. 进入扩展目录
cd appinventor-sources/appinventor/extensions

# 3. 创建扩展项目
ant create-extension -Dext.name=HelloWorld

# 4. 编译
ant extensions
```

## 八、常见问题

### Q1：扩展提示"未知组件"？

**解决方案**：
1. 确认 `.aix` 文件已正确导入
2. 清除浏览器缓存
3. 尝试使用Chrome浏览器
4. 检查是否为最新版本AI2

### Q2：扩展申请权限失败？

**解决方案**：
```blocks
// 在Screen属性中手动开启权限
设置 Screen1.Permissions:
├── BLE
├── Camera
├── Location
└── Storage
```

### Q3：扩展与组件冲突？

**解决方案**：
1. 避免使用同名的扩展和内置组件
2. 更新到最新版本扩展
