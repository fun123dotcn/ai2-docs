## 一、什么是 App Inventor 2？

App Inventor 2（简称AI2）是由麻省理工学院（MIT）开发的一款**在线可视化编程工具**，让用户可以通过拖拽积木块的方式，无需编写代码即可开发 Android 应用。

**App Inventor 2 的核心优势**：

| 优势 | 说明 |
|------|------|
| 🧩 可视化编程 | 拖拽积木块，无需写代码 |
| 🌐 完全在线 | 浏览器即可开发，有网就能用 |
| 📱 即时预览 | 手机扫描二维码实时查看效果 |
| 🎓 零门槛入门 | 适合编程初学者 |
| 💪 功能强大 | 能开发真实商业应用 |

1. 登录后，点击左上角 **"新建项目"** 按钮

```
❌ 我的第一个应用（避免中文）
❌ app 1 2 3（避免特殊字符）
```

| 组件类别 | 组件名称 | 用途 | 难度 |

### 4.2 设计一个简单界面

让我们设计一个"问候应用"的界面：

**目标**：用户输入名字，点击按钮后显示问候语

**组件清单**：

```
✅ TextBox1（文本输入框）- 让用户输入名字
✅ Button1（按钮）- 点击触发问候
✅ Label1（标签）- 显示问候语
✅ HorizontalArrangement1（水平布局）- 排列输入框和按钮
```

**操作步骤**：

1. 拖拽 **TextBox** 到界面
2. 拖拽 **Button** 到 TextBox 旁边
3. 拖拽 **Label** 到按钮下方
4. 修改各组件属性：
   - TextBox1.Width = Fill parent
   - Button1.Text = "打招呼"
   - Label1.Text = "请输入你的名字"
   - Label1.FontSize = 24

```
💡 提示：组件属性可以在右侧面板修改，常用属性包括：
   - Width（宽度）
   - Height（高度）
   - Text（文本内容）
   - BackgroundColor（背景颜色）
   - FontSize（字体大小）
```

### 4.3 界面布局技巧

**技巧1：使用布局组件**

| 布局类型 | 效果 | 适用场景 |
|---------|------|---------|
| HorizontalArrangement | 水平排列 | 按钮组、表单项 |
| VerticalArrangement | 垂直排列 | 列表、表单 |
| TableArrangement | 表格排列 | 对齐的表单 |

**技巧2：屏幕自适应**

```blocks
// 设置组件宽度为"填充父级"
设置 TextBox1.Width = "填充父级"

// 设置组件高度为"自动"

```

**技巧3：组件层级管理**

- 可以拖拽组件到其他组件内部，形成嵌套关系
- 右键组件可查看/修改层级
- 合理嵌套使界面更清晰

## 五、逻辑编写

### 5.1 进入逻辑编辑模式

**操作**：点击右下角的 **"逻辑"** 按钮，进入积木编辑器

```
界面编辑模式 ←→ 逻辑编辑模式
   [设计]          [逻辑]
```

### 5.2 积木块基础

**App Inventor 2 的积木块分类**：

| 类别 | 颜色 | 示例 |
|------|------|------|
| 控制 | 橙色 | if/then、for each、while |
| 逻辑 | 绿色 | true、false、and、or |
| 数学 | 蓝色 | +、-、×、÷、random |
| 文本 | 粉色 | 连接、获取子文本 |
| 列表 | 青色 | 创建列表、获取列表项 |
| 颜色 | 彩色 | 颜色选择器 |
| 变量 | 橙色 | 全局变量、局部变量 |
| 过程 | 深绿色 | 定义过程、调用过程 |

### 5.3 编写问候应用的逻辑

**目标**：点击按钮后，Label 显示 "你好，XXX！"

**积木块代码**：

```
当 Button1.点击时
├── 执行 Label1.文本 = 连接("你好，", TextBox1.文本)
└── 执行 Label1.文本 = 连接(Label1.文本, "！")
```

**等价的中文描述**：

```
当 用户点击"打招呼"按钮时
├── 把 Label（标签）的显示文本改成：连接("你好，" 和 TextBox（文本框）的内容）
└── 再把文本改成：连接(上面的结果 和 "！")
```

### 5.4 常用逻辑模式

**模式1：条件判断**

```
如果 TextBox1.文本 ≠ ""
则 显示问候
否则 显示"请输入名字"
```

**模式2：列表遍历**

```
for each 项目 in 列表
    处理每个项目
```

**模式3：事件响应**

```
当 组件.事件发生时
    执行相应操作
```

### 5.5 变量使用

**全局变量**：在整个应用生命周期内可用

```blocks
// 创建全局变量
初始化全局 计数 = 0

// 使用变量
当 Button1.点击时
    设置 计数 = 全局 计数 + 1
    显示 计数
```

**局部变量**：只在特定代码块内可用

```blocks
执行 with (结果 = ...)
    在此范围内使用结果
结束 with
```

## 六、测试与调试

### 6.1 实时预览（推荐）

**优势**：
- 🚀 即时看到修改效果
- 📱 使用真机测试，更准确
- 🔄 支持热更新

**连接方式**：

| 方式 | 步骤 | 适用场景 |
|------|------|---------|
| 扫码连接 | 1. 打开手机AI Companion<br>2. 扫描网页二维码 | 最简单 |
| 输入连接码 | 1. 打开手机AI Companion<br>2. 输入网页显示的6位代码 | 无扫码功能时 |
| USB连接 | 1. 手机开启开发者模式<br>2. 连接电脑 | 网络不稳定时 |

**扫码连接步骤**：

1. 确保手机和电脑在同一WiFi网络
2. 在电脑端点击 **"连接" → "AI Companion"**
3. 打开手机 App Inventor 2 Companion
4. 手机扫描电脑屏幕上的二维码
5. 连接成功后即可预览

```
💡 提示：如果扫码失败，确保：
   - 手机和电脑在同一网络
   - 手机已安装 MIT AI2 Companion
   - 二维码清晰可见
```

### 6.2 使用模拟器测试

**适用场景**：没有 Android 手机时

**操作步骤**：

1. 点击 **"连接" → "模拟器"**
2. 等待模拟器启动（约1-2分钟）
3. 模拟器自动连接开发环境

```
⚠️ 注意：模拟器较慢，建议仅用于无手机时临时测试
```

### 6.3 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 无法连接 | 网络不通 | 检查WiFi是否同一网络 |
| 连接后闪退 | 组件配置错误 | 检查属性设置 |
| 扫码失败 | 二维码过期 | 刷新页面重新生成 |
| 模拟器无法启动 | 端口被占用 | 重启电脑或更换端口 |

### 6.4 调试技巧

**技巧1：使用 SnackBar 显示信息**

```blocks
当 Button1.点击时
    调用 Screen1.显示提示信息(消息 = "调试信息")
```

**技巧2：使用日志**

```blocks
当 Button1.点击时
    调用 调试日志.追加(日志内容 = "按钮被点击")
```

**技巧3：分步执行**

```blocks
当 Button1.点击时
    执行 Label1.文本 = "第1步"
    等待 1 秒
    执行 Label1.文本 = "第2步"
```

## 七、编译打包APK

### 7.1 打包当前应用

**操作步骤**：

1. 点击顶部菜单 **"构建"**
2. 选择 **"应用（提供二维码下载）"**
3. 等待编译完成（约1-3分钟）
4. 编译成功后显示二维码，用手机扫码下载

**或者选择其他打包方式**：

| 选项 | 说明 | 适用场景 |
|------|------|---------|
| 应用（二维码） | 生成二维码扫码下载 | 快速测试 |
| 应用（仅 APK） | 生成 APK 文件下载 | 分享给朋友 |
| 应用（调试 APK） | 带调试信息的 APK | 开发者测试 |

### 7.2 编译原理简介

**App Inventor 2 的编译流程**：

```
┌─────────────────┐
│   积木块代码    │  ← 用户在浏览器中拖拽的积木
└────────┬────────┘
         │ 转换
         ▼
┌─────────────────┐
│   YAIL 代码     │  ← 积木转化的 Scheme 代码
└────────┬────────┘
         │ 编译
         ▼
┌─────────────────┐
│   Android 源码  │  ← 生成 Java/Android 代码
└────────┬────────┘
         │ 构建
         ▼
┌─────────────────┐
│   APK 文件      │  ← 最终安装包
└─────────────────┘
```

### 7.3 编译时间说明

| 项目复杂度 | 预计时间 |
|-----------|---------|
| 简单应用（&lt;10个组件） | 30秒 - 1分钟 |
| 中等应用（10-50个组件） | 1 - 2分钟 |
| 复杂应用（&gt;50个组件） | 2 - 5分钟 |
| 含扩展的应用 | 可能更长 |

```
💡 提示：编译服务器位于 MIT（美国），国内访问可能较慢
   中文网 (fun123.cn) 提供了国内优化服务器，编译更快
```

## 八、应用签名与发布

### 8.1 什么是应用签名？

**应用签名**是开发者的数字证书，用于证明应用是由你开发的。

| 签名类型 | 说明 | 适用场景 |
|---------|------|---------|
| Debug 签名 | 临时签名，仅测试用 | 开发阶段 |
| Release 签名 | 正式签名，可发布 | 上架应用商店 |

### 8.2 Debug APK vs Release APK

| 特性 | Debug APK | Release APK |
|------|-----------|-------------|
| 签名 | 自动临时签名 | 需要自行配置签名 |
| 大小 | 较大（含调试信息） | 较小（已优化） |
| 速度 | 稍慢 | 已优化 |
| 用途 | 测试 | 发布 |

### 8.3 发布到 Google Play

**前置要求**：

| 要求 | 说明 |
|------|------|
| Google 开发者账号 | 注册费 $25（一次性） |
| 应用图标 | PNG 格式，建议 512x512 |
| 应用截图 | 手机截图若干张 |
| 隐私政策 | 如果应用收集数据则必须 |
| 应用签名 | Release 签名 |

**发布步骤**：

1. **准备应用资源**
   ```
   ✅ 应用图标 (512x512 PNG)
   ✅ 应用截图 (至少2张)
   ✅ 隐私政策 URL（可选）
   ✅ 应用描述文案
   ```

2. **构建 Release APK**
   ```bash
   在 App Inventor 2 中：
   构建 → 应用（仅 APK）
   ```

3. **注册 Google Play 开发者账号**
   - 访问 Google Play Console
   - 支付 $25 注册费
   - 完成开发者信息填写

4. **创建应用并填写信息**
   - 应用名称
   - 应用说明
   - 类别选择
   - 上传截图和图标
   - 设置价格（免费/付费）

5. **上传 APK**
   - 上传 Release APK
   - 配置应用签名
   - 设置内容分级

6. **提交审核**
   - 检查所有必填项
   - 点击"发布"
   - 等待 Google 审核（通常 1-7 天）

```
⚠️ 重要提示：
   - Google Play 需要海外网络访问
   - 国内用户可考虑其他应用商店
   - 上架前务必阅读 Google Play 政策
```

### 8.4 发布到国内应用商店

**国内常用应用商店**：

| 商店 | 特点 | 注册要求 |
|------|------|---------|
| 应用宝 | 腾讯系，用户量大 | 企业或个体工商户 |
| 华为应用市场 | 华为手机预装 | 企业账号 |
| 小米应用商店 | MIUI 预装 | 企业账号 |
| 豌豆荚 | 曾经的主流 | 已停止个人开发者 |
| 酷安 | 开发者社区 | 个人可发布 |

**国内发布注意事项**：

1. **实名认证**
   - 个人开发者需实名认证
   - 企业需营业执照

2. **应用审核**
   - 国内审核较严格
   - 避免涉及敏感内容
   - 需要隐私政策

3. **加固和优化**
   - 建议使用第三方加固服务
   - 优化 APK 大小

### 8.5 直接分享 APK

**最简单的方式**：通过二维码分享 APK

1. 在 App Inventor 2 中构建 APK
2. 生成二维码
3. 扫码下载安装

**注意事项**：

```
⚠️ 安装限制：
   - Android 8.0+ 需要开启"未知来源"权限
   - iOS 无法直接安装 APK（需要转换工具）
   - 建议提供应用更新说明
```

**文件说明**：


## 十一、常见问题FAQ

### Q1：App Inventor 2 开发的应用能上架应用商店吗？

**答**：完全可以。

App Inventor 2 开发的是标准的 Android APK，理论上可以上架任何 Android 应用商店，包括 Google Play、华为应用市场、应用宝等。

**注意事项**：
- 需要 Release 签名版本
- 遵守各应用商店的政策
- 部分商店对个人开发者有限制

### Q2：编译失败怎么办？

**答**：常见原因和解决方案：

| 错误类型 | 可能原因 | 解决方案 |
|---------|---------|---------|
| 组件错误 | 组件未正确配置 | 检查组件属性 |
| 逻辑错误 | 积木块连接错误 | 重新拼接积木 |
| 扩展冲突 | 多个扩展不兼容 | 移除多余扩展 |
| 服务器错误 | 编译服务故障 | 稍后重试 |

### Q3：如何让应用在后台运行？

**答**：使用后台任务组件。

```blocks
当 Screen1.返回时
    启动后台任务(任务名称="保存数据")
```

### Q4：应用体积太大怎么办？

**答**：优化建议：

| 优化项 | 方法 |
|-------|------|
| 图片压缩 | 使用压缩后的图片 |
| 移除不需要的组件 | 只保留必要组件 |
| 减少扩展使用 | 避免使用过多扩展 |
| 使用 Web 接口 | 替代本地功能 |

### Q5：如何申请应用权限？

**答**：在组件属性中开启。

| 权限 | 对应组件 | 属性位置 |
|------|---------|---------|
| 网络权限 | Web 组件 | 自动获取 |
| 相机权限 | Camera | 在 Screen1 属性中开启 |
| 位置权限 | LocationSensor | 在 Screen1 属性中开启 |
| 存储权限 | File 组件 | 在 Screen1 属性中开启 |
